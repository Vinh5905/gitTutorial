git commit được sử dụng để tạo một "commit," tức là một phiên bản (snapshot) của mã nguồn của bạn tại một thời điểm cụ thể. Mỗi commit bao gồm các thay đổi mới được thực hiện trên mã nguồn từ lần commit trước đó cùng với một thông điệp mô tả về những thay đổi đó. Cú pháp cơ bản của lệnh git commit là:

Lệnh git add cho phép bạn chọn cẩn thận những thay đổi cụ thể mà bạn muốn đưa vào commit. Sau khi bạn đã thêm các thay đổi vào vùng chờ, bạn có thể sử dụng git commit để tạo một commit mới với các thay đổi này. Điều này cho phép bạn tạo ra các commit chính xác với những thay đổi bạn muốn bao gồm, giúp quản lý lịch sử phiên bản của mã nguồn một cách cẩn thận.
